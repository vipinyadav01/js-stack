# Supported Technologies

JS-Stack CLI supports a comprehensive range of modern JavaScript technologies, allowing you to build any type of application - from simple static sites to complex enterprise applications.

## Databases

### PostgreSQL üêò
- **Type**: Relational (SQL)
- **ORMs**: Prisma, Sequelize, TypeORM
- **Features**: 
  - ACID compliant
  - Advanced features (JSON, arrays, full-text search)
  - Excellent performance
  - Open source
- **Best For**: Production applications, complex queries
- **Example**: `--database postgres`

### MySQL üê¨
- **Type**: Relational (SQL)
- **ORMs**: Prisma, Sequelize, TypeORM
- **Features**: 
  - Widely used
  - Stable and reliable
  - Good performance
  - Large community
- **Best For**: Traditional web applications
- **Example**: `--database mysql`

### SQLite üóÑÔ∏è
- **Type**: Relational (SQL)
- **ORMs**: Prisma, Sequelize, TypeORM
- **Features**: 
  - File-based (no server needed)
  - Zero configuration
  - Perfect for development
  - Lightweight
- **Best For**: Development, prototypes, small applications
- **Example**: `--database sqlite`

### MongoDB üçÉ
- **Type**: Document (NoSQL)
- **ODM**: Mongoose (only compatible ORM)
- **Features**: 
  - Flexible schema
  - Horizontal scaling
  - JSON-like documents
  - Fast queries
- **Important**: MongoDB only works with Mongoose ORM
- **Best For**: Content management, real-time apps, flexible schemas
- **Example**: `--database mongodb --orm mongoose`

## ORMs & ODMs

### Prisma ‚ö°
- **Supports**: PostgreSQL, MySQL, SQLite, MongoDB
- **Version**: Prisma 5+
- **Features**: 
  - JavaScript database client
  - Auto-generated TypeScript types
  - Migrations system
  - Excellent developer experience
  - Prisma Studio (database GUI)
  - Query optimization
- **Best For**: TypeScript projects, modern applications
- **Example**: `--orm prisma`

### Sequelize üî∑
- **Supports**: PostgreSQL, MySQL, SQLite
- **Version**: Sequelize 6+
- **Features**: 
  - Mature and stable
  - Feature-rich
  - Associations and relations
  - Transactions
  - Hooks and lifecycle events
- **Best For**: Traditional applications, complex relationships
- **Example**: `--orm sequelize`

### TypeORM üìò
- **Supports**: PostgreSQL, MySQL, SQLite
- **Version**: TypeORM 0.3+
- **Features**: 
  - Decorator-based syntax
  - JavaScript-first
  - Active Record and Data Mapper patterns
  - Migrations
  - Works great with NestJS
- **Best For**: JavaScript projects, NestJS applications
- **Example**: `--orm typeorm`

### Mongoose ü¶´
- **Supports**: MongoDB only
- **Version**: Mongoose 7+
- **Features**: 
  - Schema validation
  - Middleware (pre/post hooks)
  - Population (joins)
  - Query building
  - Model methods
- **Important**: Only ORM that works with MongoDB
- **Best For**: MongoDB applications
- **Example**: `--orm mongoose` (requires `--database mongodb`)

## Authentication

### JWT üîë
- **Type**: Token-based authentication
- **Features**: 
  - Stateless (no server-side sessions)
  - Scalable
  - Works with all frameworks
  - Simple implementation
  - Self-contained tokens
- **Best For**: APIs, microservices, stateless applications
- **Example**: `--auth jwt`

### Passport üõÇ
- **Type**: Strategy-based authentication middleware
- **Features**: 
  - Multiple strategies (local, OAuth, JWT, etc.)
  - Flexible and extensible
  - Works with Express, Koa, NestJS
  - Large strategy ecosystem
- **Note**: Not compatible with Fastify
- **Best For**: Complex authentication needs, multiple providers
- **Example**: `--auth passport`

### Auth0 üîê
- **Type**: Third-party authentication service
- **Features**: 
  - Enterprise-ready
  - Social login (Google, GitHub, etc.)
  - Multi-factor authentication
  - User management dashboard
  - Works with all frameworks
- **Best For**: Applications requiring enterprise auth features
- **Example**: `--auth auth0`

### OAuth üîì
- **Type**: Standard OAuth 2.0 protocol
- **Features**: 
  - Social login support
  - Industry standard
  - Multiple providers
  - Secure token exchange
- **Best For**: Social login, third-party integrations
- **Example**: `--auth oauth`

### Better Auth üîí
- **Type**: Modern authentication library
- **Version**: Latest
- **Features**: 
  - JavaScript (TypeScript-first)
  - Next.js optimized
  - Built-in session management
  - OAuth providers included
  - Passwordless authentication
  - Modern API design
- **Best For**: Next.js applications, JavaScript projects
- **Example**: `--auth better-auth`

## Compatibility Matrix

### Database ‚Üî ORM Compatibility

#### MongoDB
- ‚úÖ **Mongoose** (only compatible ORM)
- ‚úÖ **Prisma** (with MongoDB adapter)
- ‚ùå Sequelize (SQL only)
- ‚ùå TypeORM (SQL only)

#### PostgreSQL / MySQL / SQLite
- ‚úÖ **Prisma** (recommended)
- ‚úÖ **Sequelize** (mature, feature-rich)
- ‚úÖ **TypeORM** (TypeScript-first)
- ‚ùå Mongoose (MongoDB only)

### Frontend ‚Üî Backend Compatibility

#### Next.js / Nuxt
- ‚úÖ **Backend: none** (they have built-in API routes)
- ‚ùå Cannot use separate backend (Express, Fastify, etc.)

#### React / Vue / Angular / Svelte
- ‚úÖ All backends work
- ‚úÖ Can use `none` for frontend-only apps

#### React Native
- ‚úÖ Requires a backend (cannot be "none")
- ‚úÖ All backends work (Express, Fastify, NestJS, etc.)

### Backend ‚Üî Auth Compatibility

#### Express
- ‚úÖ All auth methods work (JWT, Passport, Auth0, OAuth, Better Auth)

#### Fastify
- ‚úÖ JWT, Auth0, OAuth, Better Auth
- ‚ùå Passport (limited support - use JWT instead)

#### NestJS
- ‚úÖ All auth methods work

#### Koa / Hapi
- ‚úÖ Most auth methods work

### Frontend ‚Üî Auth Compatibility

#### Next.js
- ‚úÖ Better Auth (recommended)
- ‚úÖ Auth0
- ‚úÖ JWT
- ‚úÖ OAuth

#### React / Vue
- ‚úÖ All auth methods work

#### Angular
- ‚úÖ Auth0, JWT, OAuth, Passport
