# CLI Commands Reference

Complete reference for all JS-Stack CLI commands, options, and flags. Use this guide to understand all available options and how to use them.

## Init Command

Create a new project:

```bash
npx create-js-stack@latest <project-name> [options]
```

### Options

#### Frontend

```bash
--frontend <framework> [<framework2> ...]
```

**Available options:**
- `react` - React with Vite
- `nextjs` - Next.js 15 (full-stack, includes API routes)
- `vue` - Vue.js 3 with Vite
- `nuxt` - Nuxt 3 (full-stack, includes server routes)
- `angular` - Angular 17+
- `svelte` - Svelte 5 with Vite
- `react-native` - React Native (mobile)
- `none` - No frontend (API-only)

**Multiple Frontends:**
You can select multiple frontends (e.g., React + Vue for a monorepo):

```bash
npx create-js-stack@latest my-app --frontend react vue
```

**Examples:**
```bash
# Single frontend
npx create-js-stack@latest my-app --frontend react

# Multiple frontends (monorepo)
npx create-js-stack@latest my-app --frontend react vue

# Next.js (full-stack, no separate backend)
npx create-js-stack@latest my-app --frontend nextjs
```

**Important Notes:**
- Next.js and Nuxt are full-stack frameworks - they include API routes, so set `--backend none`
- React Native requires a backend (cannot be "none")
- Multiple frontends work best with Turborepo addon

#### Backend

```bash
--backend <framework>
```

**Available options:**
- `express` - Express.js (most popular)
- `fastify` - Fastify (high performance)
- `koa` - Koa.js (modern async/await)
- `hapi` - Hapi.js (plugin-rich)
- `nestjs` - NestJS (enterprise-ready)
- `none` - No backend (frontend-only or full-stack framework)

**Examples:**
```bash
# Express backend
npx create-js-stack@latest my-app --backend express

# Fastify (high performance)
npx create-js-stack@latest my-app --backend fastify

# NestJS (enterprise)
npx create-js-stack@latest my-app --backend nestjs

# No backend (for Next.js/Nuxt or frontend-only)
npx create-js-stack@latest my-app --backend none
```

**Important Notes:**
- Use `none` with Next.js or Nuxt (they have built-in API routes)
- React Native requires a backend (cannot be "none")
- Fastify has limited Passport.js support (use JWT instead)

#### Database

```bash
--database <database>
```

**Available options:**
- `postgres` - PostgreSQL (recommended for production)
- `mysql` - MySQL
- `sqlite` - SQLite (great for development)
- `mongodb` - MongoDB (NoSQL)
- `none` - No database

**Examples:**
```bash
# PostgreSQL (production)
npx create-js-stack@latest my-app --database postgres

# SQLite (development/prototyping)
npx create-js-stack@latest my-app --database sqlite

# MongoDB (NoSQL)
npx create-js-stack@latest my-app --database mongodb
```

**Important Notes:**
- PostgreSQL and MySQL require a running database server
- SQLite is file-based (no server needed) - perfect for development
- MongoDB requires Mongoose ORM (automatically selected)

#### ORM

```bash
--orm <orm>
```

**Available options:**
- `prisma` - Prisma (recommended, JavaScript)
- `sequelize` - Sequelize (mature, feature-rich)
- `typeorm` - TypeORM (JavaScript-first, decorators)
- `mongoose` - Mongoose (MongoDB only)
- `none` - No ORM (raw queries)

**Examples:**
```bash
# Prisma (modern, JavaScript)
npx create-js-stack@latest my-app --orm prisma

# TypeORM (works great with NestJS)
npx create-js-stack@latest my-app --orm typeorm

# Mongoose (for MongoDB)
npx create-js-stack@latest my-app --orm mongoose --database mongodb
```

**Compatibility:**
- **MongoDB**: Only Mongoose or Prisma
- **PostgreSQL/MySQL/SQLite**: Prisma, Sequelize, or TypeORM
- The CLI automatically adjusts ORM based on database selection

#### Authentication

```bash
--auth <method>
```

**Available options:**
- `jwt` - JSON Web Tokens (stateless, simple)
- `passport` - Passport.js (strategy-based, flexible)
- `auth0` - Auth0 (third-party service, enterprise)
- `oauth` - OAuth 2.0 (social login)
- `better-auth` - Better Auth (modern, Next.js optimized)
- `none` - No authentication

**Examples:**
```bash
# JWT (simple, stateless)
npx create-js-stack@latest my-app --auth jwt

# Better Auth (Next.js recommended)
npx create-js-stack@latest my-app --auth better-auth

# Passport (Express/NestJS)
npx create-js-stack@latest my-app --auth passport
```

**Compatibility Notes:**
- **Fastify**: Use JWT instead of Passport (limited support)
- **Next.js**: Better Auth is recommended
- **Express/NestJS**: All methods work

#### Package Manager

```bash
--package-manager <manager>
```

**Available options:**
- `npm` (default)
- `yarn`
- `pnpm`
- `bun`

**Example:**
```bash
npx create-js-stack@latest my-app --package-manager pnpm
```

#### Addons

```bash
--addons <addon1,addon2,...>
```

**Available options:**
- `docker` - Docker configuration (Dockerfile + docker-compose.yml)
- `testing` - Testing setup (Jest/Vitest + Testing Library)
- `biome` - Biome linter & formatter (fast alternative to ESLint+Prettier)
- `turborepo` - Turborepo monorepo setup (for multi-package projects)

**Examples:**
```bash
# Single addon
npx create-js-stack@latest my-app --addons docker

# Multiple addons
npx create-js-stack@latest my-app --addons docker,testing,biome

# Monorepo setup
npx create-js-stack@latest my-app --addons turborepo,testing
```

**Recommendations:**
- Always include `testing` for production apps
- Use `biome` for faster linting/formatting
- Use `docker` for containerized deployments
- Use `turborepo` for monorepos or multiple frontends

#### Git

```bash
--git
```

Initialize a git repository in the project.

```bash
--no-git
```

Skip git initialization.

#### Install Dependencies

```bash
--install
```

Automatically install dependencies after project creation.

```bash
--no-install
```

Skip dependency installation.

#### Other Options

##### Quick Start Flag

```bash
--yes
```

Skip all prompts and use defaults or provided flags. The project name is still customizable as the first argument.

**Example:**
```bash
npx create-js-stack@latest my-custom-name --yes
```

##### Dry Run

```bash
--dry-run
```

Preview what would be generated without creating any files. Useful for testing configurations.

**Example:**
```bash
npx create-js-stack@latest my-app --frontend react --dry-run
```

##### Verbose Output

```bash
--verbose
```

Show detailed output during generation, including all file operations and template processing.

**Example:**
```bash
npx create-js-stack@latest my-app --verbose
```

##### Preset Configuration

```bash
--preset <preset-name>
```

Use a predefined preset configuration. See available presets:

- `nextjs-app` - Next.js full-stack application
- `express-api` - Express REST API
- `mern-stack` - MERN stack (MongoDB + Express + React)
- `nestjs-api` - NestJS enterprise API
- `vue-app` - Vue.js application
- `nuxt-app` - Nuxt full-stack application
- `mobile-app` - React Native mobile app
- `optimal-monorepo` - Optimal monorepo setup
- `optimal-fullstack` - Optimal full-stack setup
- `fastify-api` - Fastify high-performance API
- `svelte-app` - Svelte application
- `angular-app` - Angular application
- `minimal-api` - Minimal Fastify API

**Example:**
```bash
npx create-js-stack@latest my-app --preset nextjs-app
```

## Examples

### Full-Stack React App

```bash
npx create-js-stack@latest my-app \
  --frontend react \
  --backend express \
  --database postgresql \
  --orm prisma \
  --auth jwt \
  --addons docker,testing \
  --git \
  --install
```

### Next.js Full-Stack

```bash
npx create-js-stack@latest my-app \
  --frontend nextjs \
  --database postgresql \
  --orm prisma \
  --auth better-auth \
  --package-manager pnpm \
  --git \
  --install
```

### Vue.js Frontend Only

```bash
npx create-js-stack@latest my-app \
  --frontend vue \
  --package-manager yarn \
  --git
```

### MongoDB with Mongoose

```bash
npx create-js-stack@latest my-app \
  --frontend react \
  --backend express \
  --database mongodb \
  --orm mongoose \
  --auth jwt
```

## Interactive Mode

If you don't provide flags, the CLI will prompt you interactively:

```bash
npx create-js-stack@latest my-app
```

You'll be asked to select:
- Frontend framework
- Backend framework
- Database
- ORM
- Authentication
- Package manager
- Addons
- Git initialization
- Dependency installation
